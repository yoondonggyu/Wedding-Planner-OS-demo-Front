Wedding OS Planner API

웨딩 OS 프런트엔드와 상호작용하는 FastAPI 백엔드 엔드포인트 모음입니다. JWT 인증, 예산/캘린더/AI 기능에 필요한 모든 요청 정보를 한 페이지에서 확인하세요.

Base URL

http://localhost:8001/api

인증 방식

로그인 성공 시 발급되는 JWT 토큰을 Authorization: Bearer <token> 헤더에 포함합니다.

메타데이터

Admin 페이지: /secret_admin
Swagger: /docs (Redoc: /redoc)

1. 인증 (Auth)

JWT 기반 로그인/회원가입 API

POST

/auth/login

이메일/비밀번호 검증 후 JWT 토큰 발급

{ "email": "user@example.com", "password": "Test@1234" }
{ "message": "login_success", "data": { "access_token": "jwt...", "token_type": "bearer", "user_id": 1, "nickname": "estar" } }
POST

/auth/signup

이메일/비밀번호/닉네임/프로필 이미지 URL을 받아 신규 사용자 생성

{ "email": "test@ws.kr", "password": "Test@1234", "password_check": "Test@1234", "nickname": "estar", "profile_image_url": "https://cdn..." }

2. 사용자 (Users)

프로필/비밀번호 관리 및 프로필 이미지 업로드

POST

/users/profile/upload

Multipart 이미지 업로드, 응답으로 CDN URL 반환

PATCH

/users/profile

닉네임 수정. 중복/길이/공백 검증

{ "nickname": "웨딩OS" }
DELETE

/users/profile

회원 탈퇴. 사용자 관련 게시글/댓글/좋아요도 함께 삭제

PUT

/users/password

기존 비밀번호 확인 후 새 비밀번호로 갱신

{ "old_password": "Old@1234", "password": "New@1234", "password_check": "New@1234" }

3. 게시판 (Posts & Comments)

웨딩 경험 공유 게시판, 좋아요 및 댓글 기능

GET

/posts

최신순 게시글 목록 조회

/posts?board_type=couple&page=1&limit=10
GET

/posts/{post_id}

상세 조회 + 댓글/좋아요 상태 포함

POST

/posts

게시글 작성 (AI 요약/태그/감성 분석은 서버 측에서 자동 수행)

{ "title": "웨딩홀 후기", "content": "...", "image_url": null, "board_type": "couple" }
PATCH

/posts/{post_id}

게시글 수정 (작성자 권한 필요)

DELETE

/posts/{post_id}

게시글 삭제 + 연관 댓글/좋아요 제거

POST

/posts/{post_id}/like

좋아요 토글, 응답으로 like_count & liked 반환

PATCH

/posts/{post_id}/view

조회수 +1

POST

/posts/upload

게시글 이미지 업로드 (jpg/png/jpeg, 5MB 이하)

GET

/posts/{post_id}/comments

댓글 목록 조회 (최신순)

POST

/posts/{post_id}/comments

댓글 작성, 서버에서 감성 분석 수행

PATCH

/posts/{post_id}/comments/{comment_id}

댓글 수정 (작성자만 허용)

DELETE

/posts/{post_id}/comments/{comment_id}

댓글 삭제

6. AI & 음성

Chat/STT/LLM 관련 API

POST

/chat

스트리밍 챗봇 (NDJSON). user_id 필수, include_context로 RAG 온/오프

POST

/chat/simple

스트리밍이 필요 없는 단발성 응답

POST

/voice/process

음성 → 텍스트 → 요약/구조화 (STT + LLM 파이프라인)

POST

/voice/response

음성 질문/명령에 대한 자연어 답변 { query, user_id }
